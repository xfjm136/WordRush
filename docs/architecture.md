# WordRush Architecture

## 定位

WordRush 面向两类场景：

- 短期冲刺：两周内快速吃透某个考试词表
- 长期巩固：四级、六级、考研等阶段性复用同一工具链

因此它不应只是一个“卡片打分器”，而应是一个词本驱动的主动回忆系统。

## 核心对象

### 1. 词本 `WordBook`

词本是第一层对象，每个词本独立管理：

- 元信息：`id`、标题、考试类型、用途描述、主题色
- 词条集合：单词、短语、搭配、例句挖空
- 进度：新词、学习中、待复习、已掌握、今日应练
- 统计：正确率、连续训练天数、预计耗时

### 2. 训练记录 `Review Session`

训练记录建议采用 append-only 形式保存：

- 词本 ID
- 词条 ID
- 训练模式
- 输入耗时
- 是否拼对
- 错误类型
- 时间戳

这让 WordRush 可以遵守 Linux 哲学：

- 原始词库保持清晰
- 训练日志可 grep、awk、导出分析
- 调度快照可重建或缓存

## 训练模式

### `scan`

- 目标：压缩首次覆盖时间
- 输入：英文 -> 中文速判
- 用法：新词首轮扫过

### `type`

- 目标：强化主动回忆和拼写
- 输入：中文/释义/例句提示 -> 键入英文
- 用法：短期冲刺的主模式

### `cloze`

- 目标：练短语、搭配和语境
- 输入：例句挖空 -> 键入短语
- 用法：四级/六级/考研都很重要

### `mistakes`

- 目标：高频回捶错词和慢词
- 输入：只取失败项和低置信项
- 用法：缩短薄弱项回路

## 调度思路

WordRush 计划采用“输入结果驱动”的调度，而不是纯人工打分：

- 是否正确
- 是否超时
- 是否属于短语/搭配
- 是否连续失败

建议默认节奏：

- 新词：`10m -> 1h -> 8h -> 1d -> 3d -> 7d`
- 失败词：`3-5` 张后立即插回
- 短语和搭配：权重高于普通单词

## 词本操作

### 导入

- 每次导入形成一个词本
- 支持为词本指定标题、考试类型和训练重点

### 合并

- 多个词本合并为一个冲刺词本
- 原词本保留，只新增一个组合词本

### 拆分

- 按标签拆分：如高频词、易错词、短语
- 按熟练度拆分：如“已掌握”和“待强化”
- 按来源拆分：如阅读、写作、真题
